## 五、验收清单

### 5.1 多实例采集

- [ ] 顾远 + 赫明的会话 JSONL 均被 collector 采集
- [ ] genark.db 中新增 agent 字段，查询可区分实例
- [ ] 赫明采集 cron 正常运行（每 30min）
- [ ] 采集日志无报错，连续运行 24h 无中断

### 5.2 拼版日报

- [ ] 每天 23:00 推送一条拼版日报到钉钉
- [ ] 日报包含顾远 + 赫明两人的独立数据段落
- [ ] 每人段落格式：会话次数/消息数、工具调用/成功率、技能变化、记忆变化
- [ ] 交汇小节在 @ 数据可用时出现
- [ ] 交汇小节在交叉验证 < 50% 时不出现（降级提示）
- [ ] 日报纯数据，不含 LLM 叙事

### 5.3 关系网络

- [ ] 交互计数准确（基于 @ 消息）
- [ ] 趋势箭头正确：↗ ≥30%增长 / → 平稳 / ↘ ≥30%下降 / — 首周无基准
- [ ] 不做协作质量评分

### 5.4 重大事件推送

- [ ] 技能单日 +5 触发单推
- [ ] 新技能获取触发单推
- [ ] 记忆新增 ≥5 条触发单推
- [ ] 成功率骤降 <50% 触发单推
- [ ] 无重大事件时不发单推

### 5.5 交叉验证

- [ ] 脚本可在 5 分钟内跑完
- [ ] 输出一致率数字 + 不一致样本
- [ ] 结论写入报告（做 / 不做 / 部分做）

### 5.6 降级

- [ ] 任一实例采集失败 → 该人段落显示「今日数据暂缺」，不阻塞其他人
- [ ] LLM 不可用时 → 不报错（Phase 2 不用 LLM，天然免疫）
- [ ] DB 写入失败 → 事务回滚，下次 cron 重采
